# Elastic Search 搜索
```text
有两类搜索
    1.queries(查询): query 可以帮我们进行全文搜索。
    2.aggregations(聚合): aggregation 可以帮我们对数据进行统计及分析。
    也可以结合 query 及 aggregation一起使用，比如我们可以先对文档进行搜索然后在进行 aggregation。
```
## 搜索文档
```text
获取所有索引文档：
    GET _search
    
获取文档并且限定返回个数：
    GET _search?size=1

对多个index进行搜索：
    GET twitter,my_index/_search?size=20

对index进行正则搜索
    GET cust*/_search

返回结果：
{
  "took" : 15,
  "timed_out" : false,
  "_shards" : {
    "total" : 1,
    "successful" : 1,
    "skipped" : 0,
    "failed" : 0
  },
  "hits" : {
    "total" : {
      "value" : 1,
      "relation" : "eq"
    },
    "max_score" : 1.0,
    "hits" : [
      {
        "_index" : "customer",
        "_type" : "_doc",
        "_id" : "1",
        "_score" : 1.0,
        "_source" : {
          "name" : "zxf"
        }
      }
    ]
  }
}

value: 表示搜索出来的总数。
his：数组表示返回的查询结果。
_source(默认以此为排序)：搜索结果的相关度，分值越高表示越匹配。默认没有 sort 的情况下，所有搜索的结果的是按照分数由大到小来进行排列的。

分页查询：
    GET twitter/_search?size=2&from=1
    或者
    GET twitter/_search
    {
      "from": 0,
      "size": 20, 
      "query": {
        "query_string": {
          "query": "*"
        }
      }
    }

    from是页码，size大小。
```
