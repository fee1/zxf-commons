# Match Query
## 查询语法格式
### 普通查询
```text
样例：
    GET indexName/_search
    {
      "query": {
        "match": {
          "FIELD": "TEXT"
        }
      }
    }
解释：
    indexName: 索引名称，必传
    query - match_all: 表查询所有，必传
    query - match: 查询匹配(会使用默认的分词器，中文将会分割每一个中文字符，英文以单词切割，分词后的查询默认条件为OR)，必传
    query - match - "FIELD": "TEXT":[
        FIELD: 查询的字段
        TEXT: 查询的内容
    ]

扩展：
    match查询默认分词后的查询结果条件都是OR
        GET indexName/_search
        {
          "query": {
            "match": {
              "address": {
                "query": "海淀区",
                "operator": "or"
              }
            }
          }
        }
    adress中包含任意一个中文字符的数据都会查找出来

    如果我们需要全匹配查询可以如下，例：
        GET indexName/_search
        {
          "query": {
            "match": {
              "address": {
                "query": "海淀区",
                "operator": "and"
              }
            }
          }
        }
    
    我们还可以使用如下方式实现同样的效果，表示至少匹配三个字符的数据才会被查询出来
        GET indexName/_search
        {
          "query": {
            "match": {
              "address": {
                "query": "海淀区",
                "minimum_should_match": 3
              }
            }
          }
        }
```
### 分页查询
```text
样例：
    GET indexName/_search
    {
      "from": 0,
      "size": 2,
      "query": {
        "match": {
          "FIELD": "TEXT"
        }
      }
    }
解释：
    indexName: 索引名称，必传
    from: 分页参数起始页，非必传
    size: 数量，非必传
    query - match: 查询匹配(会使用默认的分词器，中文将会分割每一个中文字符，英文以单词切割，分词后的查询默认条件为OR)，必传
    query - match - "FIELD": "TEXT":[
        FIELD: 查询的字段
        TEXT: 查询的内容
    ]
```
### 设置查询返回的字段
```text
样例：
    GET indexName/_search
    {
      "_source": ["city", "message"], 
      "from": 0,
      "size": 2,
      "query": {
        "match": {
          "FIELD": "TEXT"
        }
      }
    }
解释：
    _source: 需要返回的字段，数组形式传参，非必传
    indexName: 索引名称，必传
    from: 分页参数起始页，非必传
    size: 数量，非必传
    query - match: 查询匹配(会使用默认的分词器，中文将会分割每一个中文字符，英文以单词切割)，必传
    query - match - "FIELD": "TEXT":[
        FIELD: 查询的字段
        TEXT: 查询的内容
    ]

注意：以上案例基于存储在mapping设置为[_source: true]的情况下使用。

[_source: false]的情况
样例：
    GET indexName/_search
    {
      "stored_fields": ["FIELD1", "FIELD2"],
      "from": 0,
      "size": 2,
      "query": {
        "match": {
          "FIELD": "TEXT"
        }
      }
    }
```
### 计数查询
```text
样例：
    GET indexName/_count
    {
      "query": {
        "match_all": {}
      }
    }
解释：
    indexName: 索引名称，必传
    query - match_all: 全查询，类似于sql的[SELECT * FROM TABLE]
```
### 根据id查询
```text
样例：
    GET indexName/_search
    {
      "query": {
        "ids": {
          "values": ["1", "2"]
        }
      }
    }
解释：
    indexName: 索引名称，必传
    query - ids - values: 数据数据，非必传，没有值查不出数据
```
### 同时查询多个Field 
```text
    很多情况下我们可能并不会只在一个Field中做查询，往往会同时查几个字段，比如搜索是商品，我们不只是会搜索商品的名称，也会检索内容中服务检索条件
的数据。
样例：
    GET indexName/_search
    {
      "query": {
        "multi_match": {
          "query": "海淀区",
          "fields": ["city", "province", "address"]
        }
      }
    }
解释：
    indexName: 索引名称，必传
    query - multi_match - query: 需要查询的内容，会分词
    query - multi_match - fields: 需要查询的字段

扩展：
    我们可以为某个字段加上更大的权重，让它在_source的数值更大，说明某条数据更符合我们的查询内容，例：
    GET indexName/_search
    {
      "query": {
        "multi_match": {
          "query": "海淀区",
          "fields": ["city", "province", "address^3"]
        }
      }
    }
    
```

## 去除_score
```text
    假如_source对我们来说不重要我们可以使用如下查询方法去掉。
GET twitter/_search
{
  "query": {
    "bool": {
      "filter": [
        {
          "term": {
            "city.keyword": "北京"
          }
        }
      ]
    }
  }
}
```
## 使用 match query 时，默认的分词匹配查询是 OR
```text
更具分词后（分为：朝、阳、区、-、老、贾），能查出所有的结果：
GET twitter/_search
{
  "query": {
    "match": {
      "user": "朝阳区-老贾"
    }
  }
}
或者
GET twitter/_search
{
  "query": {
    "match": {
      "user": {
        "query": "朝阳区-老贾",
        "operator": "or"
      }
    }
  }
}

我们改成AND以后就只查user为朝阳区-老贾的结果：
GET twitter/_search
{
  "query": {
    "match": {
      "user": {
        "query": "朝阳区-老贾",
        "operator": "and"
      }
    }
  }
}

必须匹配成五个字符中的三个：
GET twitter/_search
{
  "query": {
    "match": {
      "user": {
        "query": "朝阳区-老贾",
        "operator": "or", 
        "minimum_should_match": 3
      }
    }
  }
}
```

## multi_match
```text
    特别指明一个专有的 field 来进行搜索，但是在很多的情况下，我们并胡知道是哪一个 field 含有这个关键词，那么在这种情况下，我们可以使用 
multi_match 来进行搜索。
GET twitter/_search
{
  "query": {
    "multi_match": {
      "query": "朝阳",
      "fields": [
        "user",
        "address^3",
        "message"
      ]
    }
  }
}
```
